# 渲染几何修正记录

日期：2026-07-14

## 背景

低机位查看器暴露出三个问题：

- court line 用固定像素粗细 `cv2.line` 画中心线，不符合真实地面线宽投影；
- 线段靠近图像边缘时，inside 判断会导致可见线过早断开；
- 没有渲染球网，低机位画面缺少关键遮挡物。

## 修正

### 地面线宽

现在每条网球场线都带有 `width_m`，默认 `0.08 m`。渲染时先在 3D 地面生成矩形 strip，再用 OpenCV 相机模型投影到图像，并用 `cv2.fillPoly` 填充。

这意味着：

- 近处线会更宽；
- 远处线会自然变窄；
- 远处线堆叠感会比固定像素线更接近真实透视；
- mask 标注也是投影后的地面线带，而不是中心线。

对于 pinhole 且无畸变的相机，地面直线投影后仍应为直线。因此该路径现在直接投影线带四个角点，形成严格四边形，避免用大量采样点拼接导致边缘轻微抖动。对于鱼眼或带畸变模型，仍保留多点采样，因为真实投影边界可能是曲线。

### 抗锯齿

渲染默认使用 `3x` supersampling：

1. 按目标分辨率的 3 倍缩放 OpenCV 内参；
2. 在高分辨率画布上投影并填充地面线带；
3. 使用 `cv2.INTER_AREA` 下采样回目标分辨率。

这样 RGB 图像中的线条边缘会更平滑，低机位远处的细线不会像直接低分辨率填充那样明显锯齿化。生成器和 viewer 都支持通过 `--supersample` 调整质量和速度。

### 边缘裁剪

可见 polyline 现在使用 `cv2.clipLine` 做图像边界裁剪。线段靠近屏幕边缘时，不再因为采样点略出界就整段断掉。

### 球网

球网现在由 3D 线段网格生成：

- y 坐标位于球场中心线 `y=0`；
- 顶部中心高度约 `0.914 m`；
- 两侧高度约 `1.07 m`；
- 使用 OpenCV 投影到图像；
- 绘制在 court line 之后，作为图像遮挡物；
- 不作为 court line mask 的正样本。

## 仍需注意

低机位 40cm 广角下，远处球场线在图像中向地平线附近压缩是正常透视现象。判断是否“不对”不能只看视觉直觉，还需要用真实 OpenCV `K/D`、真实安装高度、真实 pitch/yaw 和真实线宽做对比。

当前默认相机仍是简化 FOV 模型。接入真实标定后，应再次检查：

- 远处线间距；
- 近处线宽；
- 球网高度和位置；
- 图像边缘畸变下的曲线形态。
